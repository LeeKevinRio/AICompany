# Backtest Protocol — 回測方法論

## 鐵律

1. **Point-in-time 資料**：任何特徵、標籤、決策只能用「當下時點已知」的資料。嚴禁未來函數（look-ahead bias）。
2. **Look-ahead 自動偵測**：必須有專門測試——例如把特徵整體往前 shift 一格後，若績效不合理地幾乎不變，判定失敗。
3. **Walk-forward 分割**：滾動的訓練 / 驗證窗，不用單次切分；明確標示樣本內（in-sample）與樣本外（out-of-sample）。
4. **成本模型**：手續費、稅費、滑價都要進回測；費率由 data-engineer 查證現行法規後寫進設定檔，不憑記憶寫死。
5. **誠實呈現**：樣本外績效不佳照實顯示；禁止調參到好看為止（每次參數調整都要記錄）。

## 標準報告欄位（缺一不可）

| 欄位 | 說明 |
| --- | --- |
| CAGR | 年化報酬率 |
| 年化波動 | 報酬標準差年化 |
| Sharpe / Sortino | 風險調整後報酬 |
| 最大回撤與回撤期間 | 幅度與起迄日期 |
| 勝率、盈虧比 | 交易層統計 |
| 交易次數、週轉率 | 成本敏感度依據 |
| Buy & Hold 對照 | 同期間被動持有基準，一律並列 |
| 樣本內 / 樣本外 | 兩段分開列，不得混在一起報 |

## 執行檢查清單

- [ ] 資料經 data-engineer 品質檢查（無缺漏、無未來日期、無重複）。
- [ ] 特徵計算視窗只向過去看；標籤與訊號的時間對齊經過驗證。
- [ ] look-ahead 偵測測試存在且通過。
- [ ] 成本模型參數有來源依據與查證日期。
- [ ] 報告含全部標準欄位與 Buy & Hold 對照。
- [ ] 參數調整歷程有紀錄（改了什麼、為什麼）。

## 紅線

- 不輸出單一目標價；結論一律機率與區間，附失效條件。
- 不用樣本外資料做任何模型 / 參數選擇。
- 不刪除或隱藏難看的回測結果。
