# creative-masters — 大師視角創意發想法

你不是在「模仿名人」，而是在強制自己用**多個經過驗證的思考框架**輪流審視同一個問題，
避免第一直覺的平庸答案。模型腦中本來就有這些大師的公開方法論，這個 skill 的作用是
**保證每次都真的用上**，而不是偶爾想到才用。

## 流程（五步，不可跳步）

### 第 1 步：定錨
先用三行寫清楚，之後所有點子都要對得上這三行：
- **目標**：這次發想要解決什麼？（賣更多？更好記？更好玩？）
- **受眾**：給誰用/看？他們現在的習慣是什麼？
- **限制**：預算、技術、時程、品牌調性的硬邊界。

### 第 2 步：選 3 位大師視角
從 `references/masters.md` 依任務類型選 **3 位**（不要全上，視角太多會稀釋）：

| 任務類型 | 建議組合 |
|---|---|
| 命名、slogan、文案 | Ogilvy ＋ Young ＋ Rubin |
| 產品功能、玩法設計 | 宮本茂 ＋ IDEO ＋ Rubin |
| 企劃案、活動、內容策略 | IDEO ＋ Ogilvy ＋ Braintrust |
| 品牌敘事、定位 | Ogilvy ＋ Rubin ＋ Young |
| 卡關重審（既有方案不滿意） | Braintrust ＋ Rubin ＋ 宮本茂 |

選完後**先讀** `references/masters.md` 中對應的三節，再開始發想。

### 第 3 步：分視角發想
每位大師視角**獨立**產出 2～3 個點子。關鍵是「保持人格差異」：
- 用該大師的核心提問開場（每節 references 都有「他會問你的問題」）。
- 三個視角的點子**允許互相矛盾**——矛盾正是價值所在，不要提前調和。
- 每個點子兩三行就好：概念一句 + 為什麼有效一句 + （可選）風險一句。

### 第 4 步：Braintrust 互評
用 Pixar Braintrust 規則讓三個視角互相拆台：
- 每個點子由**另外兩個視角**各給一條 note。
- Note 只指出問題、不給解法（「這個名字三秒內記不住」而非「改成XX」）。
- 對事不對人：批評點子本身，禁止「這個方向不錯喔」式的空泛互捧。

### 第 5 步：合成與減法
- 從互評倖存的元素合成 1～2 個**最終提案**（可以混血：A 的概念＋C 的表達）。
- 最後用 Rubin 減法檢查一輪：**還能拿掉什麼？** 拿掉後更強的東西才是成品。
- 誠實標注：哪些是穩健牌、哪些是險牌（高報酬高風險），讓決策者選。

## 產出格式

```
## 定錨
（目標／受眾／限制，三行）

## 三視角發想
### 視角一：<大師名>（核心提問：…）
- 點子 A：…
### 視角二 / 三：（同上）

## Braintrust 互評
- 點子 A ← <視角二>：note…  ← <視角三>：note…

## 最終提案
1. **穩健牌**：…（來源：哪些點子的混血、為什麼）
2. **險牌**：…（風險與報酬）

## 被捨棄的與原因
（一兩行，讓決策者知道你丟掉了什麼——這常引出更好的追問）
```

## 邊界與誠實條款

- 這些是**公開方法論的蒸餾**（思想不受版權保護），不是任何人的授權背書；
  不要在對外文案中宣稱「由 XX 大師操刀」。
- 不逐字引用任何書籍段落；用自己的話重述原則。
- 若任務極小（例如只是取一個變數名），直接答就好，不用全流程——
  這個 skill 的成本要花在值得的發想上。
