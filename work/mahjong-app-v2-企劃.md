# 麻將記分 App v2 功能企劃

> 產品代號：**MahjongScore v2**
> 企劃人：creative-lead
> 日期：2026-06-28
> 現況版本參考：`apps/web/` MVP（React + TypeScript + Vite + PWA，localStorage）

---

## 0. 企劃背景與目標

### 現況痛點

v1 是一個單頁表單：牌局管理（新增/切換/刪除）、設定、逐局輸入、明細、結算全擠在同一個捲動頁面上。一場打下來，使用者要一直上下捲動；多場歷史資料沒有瀏覽入口；玩家跨場的統計也沒有。視覺上更像是開發者工具，而不是「打完牌拿出來用一下」的個人 app。

### v2 目標

1. **App 感**：開啟有品牌入場感（Splash），操作有明確頁籤結構，不像表單。
2. **導覽清晰**：功能拆進各自頁籤，每個頁籤有單一職責。
3. **歷史可見**：不只這場，能看到多場、各玩家的輸贏走勢。
4. **分數曲線**：折線圖讓「一場的走勢故事」一眼可見。
5. **延伸價值**：趣味統計、分享、快速輸入等，讓工具從「記帳」升級成「玩後回味」。

### 受眾

CEO 個人使用，偶爾給牌友看。使用情境：打牌現場（手機）、打完後回顧（手機或桌機）。操作頻率：打一場算一場，隨手記錄。

## 1. 資訊架構（IA）

### 1-1. 整體頁面流程

```
App 啟動
  │
  ▼
[Splash Screen]  ──── 1.5～2 秒品牌動畫 ────►
  │
  ▼
[主畫面 — 底部 Tab Bar]
  ├── Tab 1：牌局  （Sessions）
  ├── Tab 2：玩家  （Players）
  ├── Tab 3：設定  （Settings）
  └── （Tab 4：統計 — v2.x 可做，先留位）

打開某一場牌局 ──►
[牌局詳情頁]（巢狀 / Push 進去）
  ├── 子頁籤 A：記局  ← 新增入口
  ├── 子頁籤 B：走勢圖
  └── 子頁籤 C：明細
```

### 1-2. Tab Bar 結構

| 序 | 頁籤名稱 | 英文代號 | 圖示方向 | 主要內容 |
|---|---|---|---|---|
| 1 | 牌局 | Sessions | 麻將牌 / 播放鍵 | 所有牌局 session 清單；新增一場的入口 |
| 2 | 玩家 | Players | 人像 | 以玩家為主軸的跨場統計與歷史 |
| 3 | 設定 | Settings | 齒輪 | 底/台金額、玩家名稱管理、全域規則 |

> 為什麼是三個？個人工具操作路徑要短，三個 tab 已涵蓋核心需求；之後若加「全域統計」再插第 4 個。

---

### 1-3. Tab 1：牌局（Sessions）

**職責：瀏覽所有場次、建立新場、進入單場記錄。**

#### 頁面內容

- 頂部：頁面標題「牌局」
- 主體：牌局卡片列表（時間倒序）
  - 每張卡片顯示：場名、日期、局數、本場冠軍（或「進行中」狀態）、各玩家總輸贏
  - 卡片右上角：「...」選單（重新命名、刪除）
- 底部浮動按鈕（FAB）：「+ 新增牌局」

#### 新增牌局流程

點 FAB → 底部 Sheet 彈出：
- 輸入場名（預設帶入「YYYY/MM/DD 場」）
- 確認使用上次的玩家設定，或進入快速設定

#### 進入單場 → 牌局詳情頁

卡片點擊 → Push 到牌局詳情頁（三個子頁籤）：

**子頁籤 A：記局**（預設顯示此頁）
- 上方：即時排名條（各玩家當前累計，帶上下箭頭變化方向）
- 中間：新增一局表單（贏家、台數、自摸/放槍/放槍者）
- 下方：「結算本場」按鈕（可選，帶入結算時間戳）

**子頁籤 B：走勢圖**
- 折線圖（詳見 Section 3）
- 下方本場統計摘要（誰胡最多局、誰自摸最多、誰放槍最多）

**子頁籤 C：明細**
- 逐局列表（與 v1 RoundList 相同，加入可展開的「每人該局 +/- 金額」）
- 支援左滑刪除

---

### 1-4. Tab 2：玩家（Players）

**職責：跨場的個人成績彙整，讓玩家能看到自己的「長期牌技」。**

#### 頁面內容

- 玩家列表：所有出現過的玩家名字（從 sessions 中彙整）
- 每個玩家卡片：跨場總輸贏、出場次數、近期走勢小圖（Sparkline）
- 點入玩家 → 玩家詳情頁：
  - 跨場累計統計（總輸贏、總局數、胡牌數、自摸數、放槍數）
  - 場次歷史列表（每場輸贏結果）
  - 跨場累計走勢折線圖（以「場」為 X 軸單位）

> 資料模型注意：v1 的 Player id 是 session 內的 'p1'~'p4'，跨場玩家需要以「名字」為識別鍵做聚合，未來若要精準識別同名不同人，需要擴充 global player registry。

---

### 1-5. Tab 3：設定（Settings）

**職責：全域預設值設定、玩家管理、規則選項。**

#### 頁面內容

**區塊 1：預設金額**
- 底（base）：數字輸入
- 每台（tai）：數字輸入
- 說明文字：「新建牌局時套用此預設，進入牌局後仍可個別調整」

**區塊 2：常用玩家（快速填入）**
- 最多 8 個常用玩家名稱（可新增、刪除、排序）
- 新增牌局時可從這裡一鍵帶入 4 人名字

**區塊 3：規則選項**（v2 先規劃，dev 視工時實作）
- 莊家/連莊自動加台（開關 + 幾台）
- 顯示「本場最終分享圖卡」按鈕開關

**區塊 4：資料管理**
- 匯出所有資料（JSON 檔）
- 清空所有資料（需二次確認）

## 2. Splash Screen（載入畫面）規劃

### 目的

讓使用者感受到「這是一個有設計感的 app」而不是網頁在讀取。建立品牌記憶點。

### 時序設計

```
T=0ms    App 啟動，Splash 全螢幕覆蓋
T=0ms    開始在背景讀取 localStorage（非同步）
T=0~500ms  品牌動畫進場（Logo / 圖示 fade in + 輕微 scale up）
T=500ms  文字漸入：「麻將記分」主標 + 「台灣麻將 · 個人記分工具」副標
T=1500ms 檢查資料是否已讀取完畢：
          ├─ 已完畢 → 直接淡出 Splash，進主畫面
          └─ 尚未完畢 → 等待完畢後再淡出（最長等 3 秒，超過則顯示「讀取中…」）
```

### 視覺元素建議（給 art-lead）

- 背景：品牌主色（建議深色調，帶東方感）
- 中央：App 圖示（麻將牌圖示或抽象設計）配品牌名稱
- 動畫：淡入 + 微縮放，不要花俏旋轉；感覺沉穩
- 出場：整體淡出（fade out），不要滑走；讓主畫面自然浮現

### 技術備註（給 dev-lead）

- Splash 是一個 React component，根據 `loaded` state 控制顯示/隱藏
- 要加 `min-display-time`（至少顯示 1.5 秒）避免資料讀取太快導致 Splash 閃過看不到
- CSS transition 即可完成淡出，不需要動畫函式庫

## 3. 分數曲線圖規劃

### 3-1. 圖表定義

**圖表類型**：折線圖（Line Chart）

**X 軸**：局次（第 1 局、第 2 局… 第 N 局），從 0（開始時各人都是 0）到最後一局結束。

**Y 軸**：累計輸贏金額（台幣，正值往上、負值往下），以 0 為基準線。

**資料線**：每位玩家一條折線，共 4 條。每個資料點是「截至該局結束後，該玩家的累計輸贏」。

**基準線**：Y=0 的水平參考線（用虛線或淡色實線呈現）。

### 3-2. 資料計算邏輯

從 session.rounds 中，對每一局 i（從 1 到 N）：
- 對 rounds[0..i-1] 執行 scoreSession，得到截至第 i 局的每人累計
- 生成 N+1 個資料點（含第 0 點，代表起始 0）

範例（4 人，5 局打完）：
```
X軸:   0    1    2    3    4    5
玩家A: [0, +300, +150, +450, +300, +600]
玩家B: [0, -100, -100, +200, +200, -100]
玩家C: [0, -100, -100, -100, -400, -100]
玩家D: [0, -100, +50, -550, -100, -400]
```

### 3-3. 互動設計

**觸控/滑鼠 Hover（重點互動）**：
- 在圖上任意位置觸碰/滑入，顯示「垂直指示線」橫跨所有折線
- 指示線旁顯示 Tooltip：列出當下局次各玩家的累計金額（名字 + 金額，正值標綠、負值標紅）

**局次篩選**（可選，v2.x）：
- 可拖曳左右端點，只看某一段局次的走勢

**圖例**：
- 圖表上方或下方列出 4 人名字各自的顏色標示
- 點擊圖例可暫時隱藏/顯示某條線

**最終局標示**：
- 最後一個資料點加上「終」小圓點 + 金額標籤（正負）

### 3-4. 技術方案建議（給 dev-lead）

- 推薦使用 **Recharts**（React 生態系，SVG-based，文件齊全，互動 Tooltip 原生支援）
- 次選：**Chart.js + react-chartjs-2**（Canvas-based，效能稍好但可讀性差）
- 資料計算邏輯（累計 delta 陣列）放在 `src/scoring/` 作為純函式，獨立於圖表元件
- 新增 pure function：`buildCumulativeTimeline(rounds, players, settings) → TimelinePoint[]`
  - `TimelinePoint = { roundIndex: number; cumulative: Record<playerId, number> }`
- 圖表元件放在 `src/components/ScoreChart.tsx`，接收 TimelinePoint[] 與 players 即可渲染

## 5. 創意延伸專區

以下挑出 6 個對「個人記錄麻將」真正有使用價值的創意，每項說清楚玩法與價值。

### 5-1. 結算分享圖卡

**玩法**：
本場結算後，App 自動生成一張 PNG 圖卡，畫面包含：場名、日期、4 人排名（名字 + 輸贏金額）、最小的折線走勢縮圖。底部帶 App 名稱浮水印「MahjongScore」。
按「分享」可直接傳到 LINE / IG。

**價值**：
打完牌最自然的動作就是「傳去群組」，圖卡替代了截圖、省去手動整理的麻煩。對 CEO 個人而言，這也是 App 自然傳播的時機（牌友看到圖卡，問「這什麼 App？」）。

**實作備註（給 dev-lead）**：
用 `html2canvas` 把結算頁渲染成 canvas 再輸出 PNG；或設計一個隱藏的「分享版面」DOM 節點，只在要分享時渲染。Web Share API 支援直接呼叫系統分享。

---

### 5-2. 趣味統計標籤（本場 MVP / 放槍王 / 自摸王）

**玩法**：
每場結束後，在走勢圖頁或結算頁下方顯示趣味標籤：
- 「本場冠軍」：累計金額最高者
- 「放槍王」：該場放槍次數最多者（次數相同取金額損失最多者）
- 「自摸王」：自摸次數最多者
- 「最慘烈一局」：某一局中，贏家或輸家的單局金額最高者

每個標籤帶小圖示，一眼就看出誰是本場話題人物。

**價值**：
打完牌大家最愛說「今天誰放槍最多次！」，讓 App 幫你自動找出答案，增加回顧的趣味性，也讓結算不只是冷冰冰的數字。

**資料模型**：現有 `Round` 資料已足夠計算，純函式新增幾個統計 helper 即可。

---

### 5-3. 即時排名變化動畫（Rank Change Animation）

**玩法**：
每次新增一局，排名條不是靜態刷新，而是帶有小動畫：
- 金額數字用「rolling number」效果（從舊值跑到新值）
- 若排名順序改變，名字條目帶滑動上/下動畫交換位置
- 贏家名字短暫亮起（例如黃色邊框閃一下）

**價值**：
把「記帳」變成「有儀式感的即時反饋」。打牌本來就有競技感，每局記完看到排名跳動，比靜態刷新更有趣。技術成本不高（CSS transition），效果顯著。

---

### 5-4. 快速輸入模式（Quick Input）

**玩法**：
在「記局」子頁籤中，提供一個「快速模式」切換鈕。開啟後，介面變成大按鈕網格：
- 4 個玩家大按鈕（選贏家）→ 點一下選定贏家
- 自摸/胡牌兩個大按鈕 → 一碰即選
- 台數用 -/+ 兩個大按鈕調整
- 如果是胡牌，再點選放槍者

整個輸入流程設計成「3~4 次點擊就完成一局記錄」，不需要 dropdown 和 select。

**價值**：
打牌現場通常手機拿在手上，不想花太多精神在 App 上。大按鈕點起來快、不容易誤觸，適合「打完一局立刻記」的使用情境。

---

### 5-5. 玩家手氣分析（近況走勢標籤）

**玩法**：
在玩家頁面，每個玩家的卡片除了長期總輸贏，還顯示「最近 N 場走勢」標籤（用顏色的 Sparkline 小圖）。
點入玩家詳情頁，可看到：
- 近 5 場場均輸贏
- 近 5 場勝率（正收益的場次佔比）
- 「最長連贏場次」/ 「最長連輸場次」紀錄

**價值**：
「最近牌運好不好」是打麻將最常聊的話題。讓數據說話，比憑感覺討論更有說服力，也讓玩家頁面有真正的瀏覽動機。

---

### 5-6. 局次備註（Round Note）

**玩法**：
新增局時，可選填一個短備註（最多 20 字），例如「連莊第三圈」「十三幺」「清一色加台」。
在明細頁，備註以小字顯示在局次資訊下方。

**價值**：
台灣麻將有很多記憶點：特殊牌型、連莊狀況、某局特別戲劇性。文字備註讓「牌局日記」更有溫度，回頭看明細不只是數字，還有當時的情境。

**資料模型**：`Round` 介面新增可選欄位 `note?: string`，向下相容（舊資料無此欄位視為空字串）。

## 6. 給 art-lead 的提示

### 視覺設計重點

**1. 整體調性**
- 台灣麻將是社交娛樂，視覺不要太商務、太冷。建議帶一點「台式溫度」——可以是有個性的深色主題（夜場感）配合少量暖金色點綴，或是乾淨的白底配紅色/綠色強調色（牌桌感）。
- 個人工具不需要過多插圖，但 Splash Screen 和空狀態（沒有牌局）需要有視覺設計。

**2. 需要設計的關鍵畫面**
- Splash Screen（最重要的品牌印象點）
- Tab Bar 圖示 × 3（含 active / inactive 狀態）
- 牌局清單卡片
- 趣味標籤圖示（放槍王、自摸王等）
- 分享圖卡版面（5-1）
- 空狀態插圖（「還沒有牌局，開始一場吧」）

**3. 色彩優先決定**
- 折線圖的 4 人顏色需要在各種背景下都清晰可辨（考慮色盲友善）
- 贏（正值）= 綠色；輸（負值）= 紅色；這個語意色彩要貫穿整個 App

**4. 互動動畫方向**
- Splash 淡入淡出：沉穩為主，不要俗氣旋轉
- Tab 切換：slide 或 fade，保持流暢
- 排名條數字變化：rolling number（參考 5-3）
- 新增成功後的微反饋：短暫的勾勾確認動畫

## 7. 給 dev-lead 的提示

### 架構調整

**1. 路由（Router）**
v1 無路由，v2 的 Tab Bar + 牌局詳情頁需要路由。建議引入 `react-router-dom` v6 或輕量的 `wouter`。
路由結構：
```
/                → 主畫面（Tab Bar，預設 Tab 1）
/sessions/:id    → 牌局詳情頁（含子頁籤）
/players         → 玩家頁
/settings        → 設定頁
```

**2. 資料模型擴充（必要）**

新增 `GlobalSettings`（全域設定，與 session 內設定分離）：
```ts
interface GlobalSettings {
  defaultBase: number;
  defaultTai: number;
  knownPlayers: string[]; // 常用玩家名稱列表
}
```

`Session` 型別建議新增：
```ts
interface Session {
  // ... 現有欄位
  endedAt?: number;  // v2.x，結算時間戳
}
```

`Round` 型別建議新增（向下相容）：
```ts
interface Round {
  // ... 現有欄位
  note?: string;    // 5-6 備註，可選
}
```

**3. 新增純函式（不影響現有 scoring.ts）**

在 `src/scoring/` 新增：
```ts
// 走勢圖資料
buildCumulativeTimeline(rounds, players, settings): TimelinePoint[]
// 趣味統計
calcSessionHighlights(rounds, players, settings): SessionHighlights
// 玩家跨場彙整
aggregatePlayerStats(sessions, playerName): PlayerStats
```

**4. 技術風險**

| 風險點 | 說明 | 建議 |
|---|---|---|
| 跨場玩家識別 | v1 用 session 內的 'p1'~'p4'，跨場需以「名字」聚合，同名不同人有歧義 | v2 先用名字做 key，接受此限制；v3 再考慮 global player UUID |
| localStorage 容量 | 場次多了之後（每場幾十局）資料量會增加，但個人工具大概率不會超過 5MB 限制 | 現階段不需要特別處理，匯出功能先上 |
| Recharts bundle size | Recharts 壓縮後約 400KB，對 PWA 首次載入有影響 | 用 dynamic import (`React.lazy`) 只在走勢圖頁面載入 |
| 動畫與 PWA 效能 | Rolling number 動畫在低端手機可能卡頓 | 動畫用 CSS transition，不用 JS animation loop；並加入 `prefers-reduced-motion` 媒體查詢 |
| `html2canvas` 分享圖卡 | 部分 iOS Safari 版本對 `html2canvas` 有 clip 問題 | 測試後若有問題，改用 SVG 生成靜態圖卡 |

**5. 建議實作順序**

```
Phase 1（骨架）: Tab Bar 導覽 + 路由 + Splash Screen
Phase 2（牌局）: Sessions 清單頁 + 牌局詳情三子頁籤（記局邏輯沿用 v1）
Phase 3（圖表）: 走勢折線圖 + Tooltip
Phase 4（玩家頁）: 跨場統計基礎版
Phase 5（體驗）: 快速輸入 + 趣味統計標籤 + 分享圖卡
```
